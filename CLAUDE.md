## Project Overview

**Bombastic (봄바스틱)** — A long-term social game for 2–10 players passing a bomb. Built with Flutter + Firebase.

Firebase project: `likelion-holycow`

## Setup

```bash
# Install dependencies and run code generation
bash setup.sh

# Firebase setup (once per local machine)
dart pub global activate flutterfire_cli
flutterfire configure --project=likelion-holycow
```

`lib/firebase_options.dart` is gitignored and must be generated locally. Place `google-services.json` in `android/app/` and `GoogleService-Info.plist` in `ios/Runner/` from Firebase Console.

## Common Commands

```bash
# Run the app
flutter run

# Code generation (freezed, riverpod, json_serializable) — run after changing any model or @riverpod annotated file
dart run build_runner build --delete-conflicting-outputs

# Watch mode for code generation during development
dart run build_runner watch --delete-conflicting-outputs

# Lint
flutter analyze

# Tests
flutter test

# Cloud Functions (from functions/)
cd functions
npm run build          # compile TypeScript
npm run serve          # build + start emulator
npm run deploy         # deploy to Firebase
```

## Architecture

### Data Flow
Unidirectional: **Firestore onSnapshot → Repository Stream → Riverpod Provider → UI**

Integrity-critical writes (bomb creation, explosions) must go through **Cloud Functions** (Callable or Firestore Trigger), not direct client writes. All timers are derived from server-side `expiresAt` timestamps to prevent client manipulation.

### Flutter App (`lib/`)

```
lib/
  main.dart                        # Firebase init, ProviderScope, MaterialApp.router
  core/
    router/app_router.dart         # GoRouter with all named routes (AppRoutes class)
    theme/app_theme.dart
    constants/app_constants.dart
  data/
    firebase/firebase_providers.dart   # FirebaseAuth, FirebaseFirestore Riverpod providers
    models/                            # freezed models: GroupModel, BombModel, MissionModel, ShopItemModel, UserModel
    repositories/                      # Stream/Future wrappers over Firestore
  features/
    auth/    — AuthGate (routing by auth state), AuthController
    home/    — group list, entry point after login
    group/   — create/join pages, lobby (waiting state)
    game/    — game page, GameController, TimerController
    mission/ — daily missions
    shop/    — item shop
    result/  — end-of-game stats page, ResultShareCard
  widgets/   — shared widgets (LoadingOverlay)
```

### Naming Conventions
- **Provider**: `xxxProvider` (auto-generated by `@riverpod`)
- **Controller**: `XxxController extends _$XxxController` (Riverpod AsyncNotifier)
- **Model**: `XxxModel` (freezed + json_serializable)
- **Page/View**: `XxxPage` (ConsumerWidget or ConsumerStatefulWidget)
- **Repository**: `XxxRepository`

### Generated Files
Files ending in `.freezed.dart` and `.g.dart` are code-generated — do not edit manually. Re-run `build_runner` after changing annotated source files.

### Cloud Functions (`functions/src/`)
TypeScript, Node 20.

- `bomb/bombExpireScheduler.ts` — scheduled every 1 min; detects `expiresAt <= now` active bombs and marks them `exploded`; also `onBombExploded` Firestore trigger for next-round logic
- `group/groupTriggers.ts` — group lifecycle triggers
- `notification/fcmSender.ts` — FCM push notifications

### Firestore Structure
```
users/{uid}
groups/{groupId}
  bombs/{bombId}     # holderUid, expiresAt, status (active|exploded|defused), round
shopItems/{itemId}
```

`GroupModel.memberUids` is the fixed ordered list — array index determines bomb-passing sequence. `GroupStatus`: `waiting → playing → finished`.
